Flytotal 参数词典

这张图不讲函数，不讲源码跳转，只讲“参数到底是什么意思”。如果你看到某个参数完全没概念，先来这里查。可以把它理解成项目里的小白版词典。
Serial Basics
1. 串口基础词

port

  • 电脑连接设备时使用的串口号。
  • Windows 常见写法是 COM3、COM4。
  • 如果填错，脚本就找不到设备。

baud

  • 串口通信速度。
  • 设备和电脑两边必须一致。
  • 你这里默认常用的是 115200。

timeout

  • 串口读取时，最多等多久。
  • 时间太大，反应显得慢。
  • 时间太小，会更频繁进入空读循环。
Track Injector
2. `track_injector` 参数怎么理解
这个脚本最像“自动点位播放器”。

控制发点节奏的参数

  • scenario：选哪条预设轨迹。
  • interval：两个点之间的时间间隔。越小，目标看起来移动越快。
  • hold-repeat：同一个点重复发几次。越大，目标越像“停住了一下”。
  • settle：轨迹发完后，额外再等多久，让系统有时间稳定下来。

控制环境和记录的参数

  • rid：先给设备一个 RID 状态，例如 OK、MISSING、SUSPICIOUS。
  • boot-wait：串口打开后先等设备启动。
  • session-file：当前测试进度写到哪个 JSON。
  • session-log-dir：时间线日志写到哪个目录。
Serial Bridge
3. `node_a_serial_bridge` 参数怎么理解
这个脚本最像“状态采集器 + 结果判定器”。

自动轮询参数

  • status-interval：多久向设备发一次 STATUS。
  • selftest-interval：多久向设备发一次 SELFTEST。
  • summary-interval：多久向设备发一次 SUMMARY。
  • 如果某个间隔设成 0，就是关闭这一类自动轮询。

结果整理参数

  • events-limit：最近事件列表最多保留几条。
  • result-history-limit：最近测试结果最多保留几条。
  • offline-timeout：多久没收到有效状态就判离线。
  • echo：是否把原始串口文本直接打印出来。
Vision Bridge
4. `vision_bridge` 参数怎么理解
这个脚本最像“摄像头监控 + 跟踪 + 抓拍器”。

输入与跟踪参数

  • source：视频源。可以是摄像头编号，也可以是视频文件路径。
  • tracker：你框完目标后，程序用哪种算法继续跟住它。
  • width / height：希望摄像头输出多大画面。
  • log-interval：锁定期间，终端多久重复打印一次状态。

抓拍与输出参数

  • capture-dir：抓拍图片存到哪里。
  • capture-log-file：抓拍记录 CSV 存到哪里。
  • capture-cooldown：两次自动抓拍至少隔多久。
  • status-file：把最新视觉状态写到哪个 JSON。
  • event-id：给抓拍文件名附加一个标签。
Session Words
5. 状态文件里常见词是什么意思

scenario_name

  • 当前跑的是哪一个场景。

point_index

  • 当前跑到第几个点。

points_sent

  • 到现在为止，一共发了多少条 TRACK 命令。

status

  • 当前处于哪个阶段，例如 IDLE、RUNNING、DONE。

rid_status

  • 当前 RID 模拟状态。

vision_locked

  • 视觉侧是否已经锁定住目标。
Firmware Core
6. 当前阶段固件主链参数怎么理解
这一节开始讲的不是 Python 工具参数，而是设备端 `main.cpp + AppConfig.h` 里真正影响风险、轨迹、云台行为的核心参数。

轨迹确认参数

  • ConfirmFrames：目标连续出现多少次才算“确认成立”。
  • 变大：更稳，不容易误判，但会慢一点。
  • 变小：更快，但更容易把偶然点也当成真目标。

轨迹丢失参数

  • LostTimeoutMs：多久没再看到目标就判定丢失。
  • RebuildGapMs：中断太久后再次出现时，是否重建新轨迹。
  • 这两个参数一起决定“轨迹是延续还是重新开始”。

云台居中与扫描参数

  • CenterPanDeg / CenterTiltDeg：云台默认中心角。
  • ScanningAmplitudeDeg：扫描时左右摆动多大。
  • ScanningPeriodDivisor：扫描快慢控制。越大越慢，越小越快。
Risk Scoring
7. 风险评分参数怎么理解
你现在最需要理解的是：风险分不是一个黑盒总分，而是多个“为什么危险”的分项累加出来的。

基础与持续性分数

  • TrackingBaseScore：只要有目标，就先给一个基础分。
  • PersistenceScorePerSeen：目标每多持续出现一次，加多少分。
  • PersistenceScoreMax：持续性分数的上限，避免无限累加。
  • ConfirmedBonusScore：目标一旦确认成立，再额外加一笔分。

身份相关分数

  • RidMatchedScore：身份匹配时给负分，意思是“更安全”。
  • RidUnknownScore：身份未知时小幅加分。
  • RidMissingScore：身份缺失时明显加分。
  • RidSuspiciousScore：身份可疑时加更高的分。

距离与运动相关分数

  • ProximityThresholdMm：距离小于它时，认为“离得太近”。
  • ProximityScore：离得太近时额外加多少分。
  • MotionAnomalySpeedThresholdMmS：速度大于它时，认为运动异常。
  • MotionAnomalyScore：运动异常时额外加多少分。

最通俗的理解

  • 有目标：先加基础分。
  • 目标持续存在：继续加分。
  • 目标被确认：再加分。
  • 身份有问题、距离太近、速度异常：继续叠加风险。
  • 最后得到一个总分，决定系统处于哪种风险级别。
Risk Thresholds
8. 风险阈值和保持窗口怎么理解
系统不是一到分数线就立刻抖动切换，而是通过“阈值 + 保持时间”让结果更稳。

风险等级阈值

  • SuspiciousThreshold：达到后进入“可疑”。
  • HighRiskThreshold：达到后进入“高风险”。
  • EventThreshold：达到后进入“事件锁定”。
  • 这三条线决定风险状态和事件是否打开。

升级 / 回落保持窗口

  • SuspiciousEnterHoldMs / HighRiskEnterHoldMs / EventEnterHoldMs：升级前要先保持多久。
  • SuspiciousExitHoldMs / HighRiskExitHoldMs / EventExitHoldMs：回落前要先保持多久。
  • 作用：减少状态来回抖动，让事件开关更稳定。
How To Read
9. 小白现在最应该先理解什么

先理解这 4 个概念

  • 什么是 active：系统当前认为“有目标”。
  • 什么是 confirmed：系统认为“这个目标不是偶然噪声”。
  • 什么是 risk_score：系统对危险程度的量化分数。
  • 什么是 event_active：系统已经正式把这次风险过程当作事件处理。
