`track_injector` 详细功能逻辑图

这张图专门对应 tools/track_injector_轨迹注入器.py。它不是画轨迹的 GUI，而是把 track_scenarios.json 里的离散点，按时间顺序通过串口发给 Node A。 你可以把它理解成“测试场景播放器 + 会话状态记录器”。图里把 main()、run_scenario()、 参数、session 字段、发点循环、输出文件都拆成了源码可对照的块。
Role

Main Parameters
3. `main()` 参数图
这些参数决定“往哪个设备发”“以什么节奏发”“把过程记到哪里”。

串口连接参数

  • --port：必填。串口号，例如 COM4。
  • --baud：波特率，默认 115200。
  • --boot-wait：串口打开后，给 ESP32 预留启动时间，默认 3.5 秒。

轨迹执行参数

  • --scenario：跑哪个场景。可选某个场景名或 all。
  • --interval：两次 TRACK 命令之间的间隔秒数，默认 0.18。
  • --hold-repeat：同一个点重复发多少次，默认 2。这个参数会直接影响“停留感”。
  • --settle：发完 TRACK,CLEAR 后等待多久，默认 1.5 秒。
  • --rid：可选。先发一条 RID,OK、RID,MISSING 或 RID,SUSPICIOUS。

文件输出参数

  • --session-file：当前会话快照 JSON，默认 captures/latest_test_session.json。
  • --session-log-dir：时间线目录，默认 captures/session_logs。
  • 这两个路径都会经过 resolve_path() 处理，支持相对路径。
Function Map
4. 每个函数到底负责什么

路径与配置

  • resolve_path()：把相对路径拼到项目根目录。
  • load_scenarios()：读取场景 JSON 文件。

状态写入

  • build_session_payload()：统一构建会话状态字典。
  • write_session_json()：先写临时文件再替换，防止半写入。

串口通信

  • send_line()：发送一行命令，附加换行并立刻 flush。
  • read_serial_background()：读设备回显并打印成 [ESP32] ...。

场景执行

  • run_scenario()：执行单个场景，是整份脚本的核心。
  • main()：负责把上面所有零件串起来。

Session Fields
8. `build_session_payload()` 里的字段说明

场景标识字段

  • scenario_name：当前正在跑的场景名。
  • scenario_description：场景说明文字。
  • scenario_index：当前是第几个场景。
  • scenario_total：本次总共会跑多少个场景。
  • rid_mode：本轮 RID 模式，若没指定就是 UNCHANGED。

进度字段

  • point_total：场景中唯一点的数量。
  • repeat_total：每个点要重复发多少次。
  • point_index：当前走到第几个点。
  • repeat_index：当前点的第几次重复。
  • points_sent：累计发了多少条 TRACK 命令。

状态与时间字段

  • status：IDLE / RUNNING / SETTLING / DONE / ALL_DONE。
  • available：只要不是 IDLE 就是 true。
  • current_x_mm / current_y_mm：当前正在发送的点坐标。
  • started_ms：本场景开始时间。
  • updated_ms：这次 payload 生成时刻。
  • finished_ms：本场景或所有场景结束时间。
Scenario Geometry
9. 场景里的“轨迹逻辑”到底是什么
这里不是连续曲线求解，而是离散点序列播放。轨迹看起来在动，是因为设备接收到一串按时间排序的点。

`left_to_right`

  • 固定 y=1800。
  • x 从负到正递增。
  • 含义：目标从左向右横穿。

`right_to_left`

  • 固定 y=1800。
  • x 从正到负递减。
  • 含义：目标从右向左横穿。

`enter_hold_leave`

  • 通常固定 x=0。
  • y 先接近，再在近点重复，再离开。
  • 含义：目标进入中心区域、短暂停留、再退出。
