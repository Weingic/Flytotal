`track_injector` 关键参数

  • scenario：跑哪个场景。
  • interval：两条 TRACK 间隔。
  • hold-repeat：每个点重发几次。
  • settle：清轨后等待时间。
  • rid：先设置的 RID 模式。

`main.cpp` 关键入口

  • setup()：创建 RadarTask、TrackingTask、CloudTask。
  • pollHostCommands()：读取主机串口命令。
  • handleHostCommand()：分发 TRACK、RID、STATUS、RESET 等。

`node_a_serial_bridge` 关键参数

  • status-interval：自动轮询 STATUS。
  • selftest-interval：自动轮询 SELFTEST。
  • summary-interval：自动轮询 SUMMARY。
  • offline-timeout：多久判离线。

SHARED FILES
4. 整条链路共享的关键文件
理解这些文件后，你就能知道不同脚本是怎么互相“看见彼此状态”的。

`latest_test_session.json`

  • 由 track_injector 写。
  • 描述当前跑到哪个场景、哪个点、当前状态是不是 RUNNING / SETTLING / DONE。

`latest_node_status.json`

  • 由 node_a_serial_bridge 写。
  • 是设备侧统一状态快照。

`latest_test_result.json`

  • 由 node_a_serial_bridge 在 session 完成时写。
  • 包含 PASS / WARN / FAIL 和对应原因。

`session_logs/*.jsonl`

  • 由 session_log_utils 统一维护。
  • 串起注入、设备状态、视觉状态、抓拍、测试结论。
