# 2026-04-21 Day5 Win 稳定性记录

## 范围
- 异常串口输入保护
- 抓拍失败 / 抓拍路径缺失保护
- 字段缺失时事件对象继续成立
- 冷却期 / 重开边界可追踪

## 本轮补强
- 主链增加空命令与超长命令拦截，避免串口脏输入继续进入分支解析。
- `VISION` / `AUDIO` / `RESET` 手工仿真命令改为统一刷新时间戳、触发位和 node cache，避免状态改了但节点缓存没更新。
- 事件状态输出新增 `capture_evidence_state`：
  - `NO_EVENT`
  - `PENDING`
  - `BOUND`
  - `MISSING_TOLERATED`
- 抓拍路径为空时继续保留事件对象，用 `NONE` 占位，不因证据链暂缺直接丢事件。

## 关键场景记录
1. 合法目标 + RID_MATCHED + VISION_LOCKED
   - 目标：确认正常链路不被稳定性补丁打坏。
   - 预期：事件按原策略工作，状态输出可读。
2. 非合作目标 + 无 RID + VISION_LOCKED
   - 目标：确认高风险事件链仍可正常升级。
   - 预期：事件对象保留，风险原因仍可解释。
3. VISION_LOST + RID_MATCHED
   - 目标：确认视觉抖动不会导致异常关键信息丢失。
   - 预期：事件生命周期日志仍可回看。
4. 抓拍失败 / capture_path 缺失
   - 目标：确认事件仍成立，证据状态变成 `MISSING_TOLERATED`。
   - 预期：不崩、不丢事件、不破坏详情页字段。
5. 冷却期内重复触发
   - 目标：确认冷却期仍阻止误重开。
   - 预期：日志可看到阻塞原因，事件不乱开。

## 当前结论
- 这轮补丁的目标不是新增能力，而是把“异常情况下也不乱”的行为固定下来。
- 单节点主链仍保持兼容；新增字段主要服务验收、答辩和排障。
