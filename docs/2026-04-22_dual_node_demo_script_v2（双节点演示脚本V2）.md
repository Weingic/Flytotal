# 2026-04-22 双节点演示脚本 V2

## 目标
- 把 `A1 / A2` 双节点系统讲成一条完整的“发现 -> 风险 -> 接力 -> 证据 -> 回放”故事线。
- 5.1 现场只按固定顺序展示，不临时改口径。

## 演示边界
- 当前版本主打“双节点可观测接力”和“事件留痕可讲清”。
- 当前版本不宣称已完成复杂跨节点轨迹拼接算法。
- 如果真实接力当天未完全稳定，优先展示：双节点在线、独立事件、handoff 字段、导出与回放。

## 演示前 3 分钟检查
1. A1、A2 bridge 都在运行。
2. `http://127.0.0.1:8765/vision_dashboard.html` 可打开。
3. dashboard 双节点区显示 A1 / A2。
4. 事件详情卡片可打开。
5. 导出列表区可读取。
6. 若为真实链路，确认当前不是 mock 模式。

## 固定演示顺序
### 1. 先讲“双节点不是单节点改名”
口播重点：
- 这是 `A1 + A2` 两个真实节点，不是单节点页面复制。
- 双节点区会同时显示：
  - 在线状态
  - 最后上报时间
  - 当前 tracks / events
  - `node_id / source_node`
  - `continuity_hint`
  - `handoff_from -> handoff_to`

### 2. 再讲“事件由谁先发现”
口播重点：
- 当目标先进入 A1 区域时，A1 先产生跟踪与风险状态。
- 事件详情里先看：
  - `event_id`
  - `node_id`
  - `source_node`
  - `risk_level`
  - `rid_status`
  - `reason_flags`

### 3. 再讲“为什么会切到 A2”
口播重点：
- 发生节点切换时，不靠口头解释，而是看字段。
- 重点解释：
  - `prev_node_id`
  - `handoff_from`
  - `handoff_to`
  - `continuity_hint`
- 推荐口播模板：
  - “这条记录当前属于 A2，上一个节点是 A1，这里能看到本次接力方向是 A1 -> A2，因此接力关系不是隐藏在代码里，而是页面、事件对象和导出里都能看到。”

### 4. 再讲“证据如何导出”
操作顺序：
1. 选中目标事件。
2. 打开事件详情。
3. 点击“导出当前事件证据 JSON”。
4. 切到“最近证据导出”区，展示刚导出的文件。

口播重点：
- 导出不是截图，而是结构化证据。
- 导出里至少能讲清：
  - 事件是谁先发现的
  - 当前由谁负责
  - 是否发生节点切换
  - 风险和身份状态是什么

### 5. 最后讲“回放与留痕”
口播重点：
- 点击导出列表中的回放按钮，可以回放当前导出证据。
- 评委如果质疑“是不是现场临时说的”，就回到导出与日志对照。

## 必讲字段
- `event_id`
- `node_id`
- `source_node`
- `prev_node_id`
- `handoff_from`
- `handoff_to`
- `continuity_hint`
- `risk_level`
- `rid_status`
- `reason_flags`
- `capture_path`
- `evidence_hash`

## 现场答辩一句话版本
- “我们不是只做了两个节点同时在线，而是把目标从 A1 到 A2 的接力关系显式保存在状态、事件对象和导出证据里，所以双节点协同是可展示、可回放、可解释的。”
